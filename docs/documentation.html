Documentation¶

Just like any scientific analysis, code is much more useful when it’s properly documented. For this project, we use Sphinx, which is arguably the most common framework to document Python projects.

Format¶

The documentation consists of source files, which can be found in the docs/source folder. It’s structured as a hierarchy of *.rst. The index.rst file is the root of the documentation, all other files that should be included must be linked from this file directly or indirectly.

The documentation in the *.rst files can be built to create a single PDF document or a tree of HTML files that can be hosted online.

reStructuredText¶

The documentation is written using reStructuredText markup language. It provides a simple way to structure documents written in plain text. For example you can make a line a section heading by underlining it with =:

Section
=======

To see how it is used it is probably easiest to just have a look a the *.rst file in the docs folder our consult the Sphinx reference on reStructuredText linked above.

Building the documentation¶

To build the HTML version of the documentation navigate to the docs subfolder and run the following command:

sphinx-build source .

Viewing the documentation¶

By default, sphinx-build generates documentation in HTML format. You can view the documentation by opening the index.html document in the docs folder.

The HTML documents in the docs folder are also hosted on the GitHub pages site simonpf.github.com/regn. The GitHub pages service simply hosts the content of the docs folder, so to update the online documentation it is sufficient to push the changed html documents to the remote repository.

Note

If you add documentation to the project that results in a new HTML file in the docs folder, you will need to add this to git and push to the central repository for it to become available on simonpf.github.com/regn.

Including Jupyter notebooks¶

We are using nbsphinx to include notebooks in this documentation. In this way we can combine all project documentation in a single document.

To include a Jupyer notebook located in the notebooks directory, two steps are necessary:

  1. You need to first create a symbolic link in the docs/notebooks directory. For the train_qrnn.ipynb notebook for example the corresponding command would be:

ln notebooks/examples/train_qrnn.ipynb docs/notebooks/examples/train_qrnn.ipynb

  1. The notebook can then be included in the documentation as follows:

.. toctree::
  :maxdepth: 1
  :caption: Examples

  notebooks/examples/train_qrnn
